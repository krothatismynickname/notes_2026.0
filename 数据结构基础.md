## 2 线性部分
### 2.1 线性表（列表）
1. 定义：有限同类型数据元素组成的有序表
2. 基础操作：创建空列表、返回列表长度、查找、插入、删除、排序
### 2.2 顺序表
1. 定义：连续存储的列表 → 物理空间连续
2. 顺序表类封装了各类操作和功能
3. 静态空间 → 上溢出/下溢出 → 动态空间
4. 对比$$\begin{cases}&递增&倍增\\累积增容时间&O(n^2)&O(n)\\分摊增容时间&O(n)&O(1)\\装填因子&100\%&50\%\end{cases}$$
5. 对比$$\begin{cases}平均复杂度&加权复杂度&独立化各操作&不准\\分摊复杂度&极限平均复杂度&连续,总体\to单次&精准\end{cases}$$
### 2.3 链表
1. 定义：链接方式存储的列表
2. 常规形式：单链表、循环链表、双链表
3. 结点：【数据 | 下一数据地址】（插入、删除便捷，但是危险）
4. 头节点：有数据的链表中指向第一个数据，无数据的链表中指向`NULL`（个人觉得`nullptr`更好）
5. 头节点的好处：不用差异化对有数据列表和空列表的数据插入和删除
6. 插入：申请个新的结点，然后`newNode -> next = nextNode -> next; lastNode -> next = newNode`
7. 删除：`T* k = nodeLast -> next; nodeLast -> next = nodeLast -> next -> next; delete k;`
### 2.4 栈
1. 单端 → 先进后出、后进先出（FILO, LIFO）
2. 用栈顶表示当前栈的端处（`top = -1`表栈空）
3. 用途：逆序输出、递归嵌套、延迟缓冲、栈式计算、括号匹配
### 2.5 队列
1. 双端 → （FIFO）
2. front → 出队当前处，rear → 入队位置
3. 顺序队列 → 假溢出 → 循环队列（错误方法：拉大队列长度，一直怼回开头）
4. 循环队列：
	1. 空：`front == rear`
	2. 满：`(rear+1)%m ==front`
	3. 队列长度：`(m+rear-front)%m`
5. 应用：分配CPU
### 2.6 补充

# 乙 · 非线性结构
## 1 树&二叉树
### 1.1 树
1. 应用于分级结构：表达式、文件目录结构、网络域名、IC设计
2. 树=根+子树
3. 术语：
	1. 父结点、子结点、边、姊妹结点
	2. 树的高度/深度
	3. 结点的度、树的度：子树个数
	4. 叶结点/内部结点
	5. 有序树、无序树
	6. 森林
### 1.2 二叉树
1. 二叉树=根+左子树+右子树
2. 法则
	1. 深度为 i 的结点至多有 $2^i$ 个
	2. 深度为 k 的树至多有 $2^{k+1}-1$ 个结点
	3. 叶结点数 = 度为2的结点数 + 1
	4. n 个结点的二叉树的深度为为 $\log_2^n$ ，向下取整
	5. 对二叉树各结点进行编号，则 `0` → 根结点，`i` 的父结点是 `(i-1)/2`，左子结点是 `2*i+1`，右子结点是 `2*i+2`
3. 存储结构：（1）按照编号 → 顺序表（2）结点类中声明数据、左右子结点
4. 遍历顺序：先序遍历、中序遍历、后序遍历、层次遍历
5. 转化
	1. 树/森林 → 二叉树：姊妹结点/各根结点 → 右子结点递归，子结点 → 左子结点递归，边 → 1左(n-1)右
	2. 二叉树 → 树/森林：右子结点递归 → 姊妹结点/其他根结点，左子结点递归 → 子结点
### 1.3 哈夫曼树
1. 权重路径长度（WPL）
2. 最优二叉树： $WPL=WPL_{min}$
3. 哈夫曼树：权重越大，深度越小

# 丙 · 查找排序
## 1 查找
### 术语
检索、关键词、平均查找长度
### 1.1 顺序查找$O(n)$
1. 要求：线性逻辑结构
2. 查找平均次数为 $\frac{n(n+1)}{2}$ ，适合小型查找表
### 1.2 二分查找$O(\log n)$
1. 要求：线性逻辑结构+线性有序存储
2. 中间 / 递归到左右两边，若递归到左>右，则找不到
### 1.3 二叉排序树查找$O(\log n)\sim O(n)$
1. 左子树（若不为空）结点 < 根结点 < 右子树（若不为空）结点
2. 左 / 右子树也是二叉排序树
3. 插入过程：小了 → 左子树，大了 → 右子树，递归这一块
4. 删除：右子树补上来
5. 中序遍历 → 有序排列的线性表
### 1.4 哈希查找$O(1)$
1. 数据 → 关键字 → 哈希函数 → 数据所存的地址
2. 冲突：关键字不同 → 地址相同
3. 克服冲突方案：构造好的函数 / 设计解决冲突的方式
4. 构造要求：地址有值域、要连续，函数要简单
5. 常见哈希函数：一次函数、数学分析、取余
6. 冲突解决：开放地址 / 用链表
## 2 排序
### 2.0 术语
排序、排序关键字、时间复杂度、稳定性
### 2.1 插入排序
直接插入排序$O(n^2)$，经过$n-1+\tau次\qquad\tau为逆序数$
### 2.2 选择排序
直接选择排序$O(n^2)$，经过$\frac{n(n-1)}{2}$次
### 2.3 交换排序
1. 冒泡排序$O(n^2)$
2. 快速排序：不稳定、空间$O(1)$，时间$O(n\log n)\sim O(n^2)$
### 2.4 归并排序（稳定）$O(n\log n)$
递归分解 → 排序 → 每次收束都要排好序
